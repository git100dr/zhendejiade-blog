## 建站方案

**工作原理**：

    本地开发网站，生成 HTML、CSS、JavaScript 文件，然后将这些文件上传到托管服务商。这些服务商会直接将这些静态文件分发给用户，速度快，安全性高，并且完全免费。

    **可选方案：**

    • **GitHub Pages:**
    	○ 优点: 与 GitHub 账户完美集成，简单易用，尤其适合程序员。你可以通过 Git 来管理和更新你的网站内容。
    	○ 如何使用: 将你的网站代码上传到一个 GitHub 仓库，然后在仓库设置中开启 GitHub Pages。你的网站地址通常是 你的用户名.github.io。你可以通过 Cloudflare 将你的免费域名指向这个地址。
    	○ 特点: 适合技术背景的用户，更新内容需要提交代码。

    • **Vercel**: 本质采用方案
    	○ 优点: 开发者友好，部署速度快，支持多种前端框架，可以和 GitHub, GitLab 等代码仓库集成。每次代码提交都会自动部署。
    	○ 如何使用: 将你的网站代码托管到 GitHub，然后通过 Vercel 官网授权连接你的 GitHub 仓库，选择你的项目，即可自动部署。
    	○ 特点: 部署流程非常流畅，功能强大，适合希望快速上线和持续迭代的用户。

    • **Netlify**:
    	○ 优点: 功能丰富，提供持续部署（CI/CD）、表单处理、无服务器函数等功能，免费套餐额度很高。
    	○ 如何使用: 类似 Vercel，将代码托管到 GitHub，然后通过 Netlify 授权连接即可。
    	○ 特点: 强大的工具链，适合需要更多高级功能的用户。

**配置 Cloudflare**：

    无论你选择哪个平台，配置 Cloudflare 都很简单。在托管服务商（如 Vercel）的后台绑定你的域名后，服务商会提供一个或多个 IP 地址或 CNAME 记录。你只需要到 Cloudflare 的 DNS 设置页面，添加相应的记录，将你的域名解析到这些地址即可。

**流程**：

    1. 注册Github
    2. 注册Vercel账号，选择模板
    3. 绑定Github Domain
    4. 本地编辑，远程推送：pyCharm
